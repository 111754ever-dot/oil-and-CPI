# B3 方案：符号约束结构识别（原理 + 内容 + 计划，先理解再动手）

> 本文件**只讲方案和原理，不写代码**。目标：让你彻底理解 B3 在做什么、为什么这么做，然后我们再动手写。

---

## 一、B3 是什么？为什么是核心？为什么感觉复杂？

- **B3 = 把 B2 估出来的"混在一起的冲击"，分离成三类有经济含义的油价冲击**（供给 / 全球总需求 / 油价特定需求），再看它们如何影响各国通胀分布。
- **它是论文的招牌**：B2 的 GIRF 只是"描述性地看到异质性"；B3 才能回答**因果问题**——"哪一类油价冲击推高了通胀尾部风险"（对应原文图 6、图 7）。
- **为什么感觉复杂**（你的直觉对）：B3 涉及"旋转 + 符号检验"的识别算法，比 B2 的"改维度"要动脑。**但**：B2 已经把最难的贝叶斯估计做完了，B3 是在 B2 的**后验结果上做后处理**——概念丰富，但代码量有限、不改估计核心。

## 二、B3 的原料：B2 的后验给了我们什么

B2 每保存一次 MCMC 抽样，我们手里就有：
- **Φ**（状态 VAR 的系数）—— 描述因子和全局变量怎么随时间演化；
- **Ω**（状态 VAR 残差的协方差矩阵，17×17）—— 这就是"混在一起的冲击"的相关结构；
- **载荷 L** —— 把因子响应投影回 64 个国别序列的桥梁。

**B3 就站在这些后验抽样上工作。**

## 三、核心原理（用大白话一步步讲）

### 3.1 "混在一起的残差" vs "干净的结构冲击"
B2 估出的是**简约式残差** ε_t（17 维），它们**彼此相关**（Ω 不是对角阵）——油价上涨那一刻，"供给中断 / 需求过热 / 恐慌囤油"混在一起，没有经济含义。

我们想要的是**结构冲击** η_t：互相独立、各有含义。两者关系：

> **ε_t = B · η_t，其中 B·B' = Ω**

矩阵 **B** 就是"把干净冲击映射成混合残差"的钥匙。找到 B，就能反推出结构冲击、算出结构脉冲响应。

### 3.2 麻烦：B 不唯一 → 需要"旋转"
满足 B·B' = Ω 的 B 有**无穷多个**。先取一个最简单的解 **P = chol(Ω)**（Cholesky 分解）；那么对任意**正交旋转矩阵 Q**（满足 Q·Q'=I），**B = P·Q 也是解**（因为 PQ·Q'P' = PP' = Ω）。

所以"识别"= **在无穷多个旋转 Q 里，挑出那个有经济含义的**。

### 3.3 怎么挑 Q：符号约束 + 接受/拒绝（RWZ 2010 算法）
用经济学给"对的 Q"设门槛——**规定三类油价冲击当期对油市变量的反应方向**：

| 油市块变量 | ①供给冲击 | ②总需求 | ③油价特定需求 |
|------------|:---------:|:-------:|:-------------:|
| 全球原油产量 | − | + | + |
| 全球经济活动(IGREA) | − | + | − |
| 实际油价 | + | + | + |

算法：**随机生成一个 Q → 算出 B=PQ 蕴含的当期反应 → 检查油市块是否满足上表 → 满足就留下，不满足就换一个 Q**。重复很多次，得到一批"合格的 B"。

### 3.4 关键设定：部分识别（只约束油市块、放开分位数因子）
- **只约束"油市块"这 3 个变量**（原油产量、IGREA、实际油价），**只识别这 3 个油价冲击**；GSCPI、GEPU 只是控制、不赋予冲击含义；
- **通胀等 12 个分位数因子完全不加约束** —— 这样**尾部（10%、90%）的反应完全由数据决定**，保住了我们最关心的"尾部异质性"。这比原文（约束中位数因子）更干净。

### 3.5 弹性约束（Kilian-Murphy：锐化识别）
纯符号约束会保留一些"短期油供给弹性大得离谱"的荒谬解。所以再加一条：**供给的短期价格弹性 ≤ 0.0258**（当期约 0.01–0.02，即短期供给几乎无弹性）。加了它，合格的 B 会少而精。
> 实现要点：弹性要在**原始单位**上算，而我们数据是标准化过的——所以算弹性时要用保存的标准差把响应"还原"回原单位。这是一个要小心的技术细节。

### 3.6 一句话原理
> **把 Ω 拆成"Cholesky × 旋转"，用油市块的符号 + 供给弹性当门槛，筛出有经济含义的旋转，从而分离出三类油价冲击；分位数因子全程不约束，尾部由数据说话。**

## 四、状态向量布局（让你看清"油市块在哪"）

B2 的状态向量 z_t 是 17 维，顺序是：

```
位置 1-4  : 10%分位数因子  [CPI IP RATE NEER]
位置 5-8  : 50%分位数因子  [CPI IP RATE NEER]
位置 9-12 : 90%分位数因子  [CPI IP RATE NEER]
位置 13-17: 全局变量       [oil_prod IGREA loil_price GSCPI GEPU]
                            └── 油市块(13-15) ──┘  └─控制─┘
```

- **符号约束只作用在位置 13、14、15**（油市块）；
- **位置 1-12（所有分位数因子）不约束** → 尾部自由；
- 位置 16、17（GSCPI、GEPU）是控制，不识别。

## 五、B3 的具体步骤（代码将来会做什么）

对**每一份 B2 后验抽样** (Φ, Ω)：
1. **Cholesky**：P = chol(Ω)。
2. **旋转搜索**：随机生成正交 Q（对随机正态矩阵做 QR 分解，服从 Haar 测度）。
3. **算当期反应**：候选 B = P·Q，读出油市块（行 13-15）的当期反应。
4. **符号 + 弹性检验**：是否满足第三节的表 + 供给弹性上界？（列可重排/变号以匹配三类冲击标签）
5. **接受/拒绝**：满足 → 保留这个 B（记下对应三类油价冲击的 3 列）；不满足 → 回第 2 步（设上限如 1000 次）。
6. **算结构脉冲响应**：用 Φ 的伴随式向前迭代 nhor 期，乘上 B → 得到**各分位数因子对三类油价冲击**的结构脉冲响应。
7. **投影到国别**：乘载荷 L → 各国的结构响应（截面异质性）。
8. **结构方差分解 FEVD**：三类油价冲击各解释了通胀不同分位数波动的多少。
9.（可选）**历史分解**：把 2021-2023 各国通胀飙升归因到三类油价冲击。

把所有后验抽样的结果汇总 → 得到结构脉冲响应的**后验分布**（中位数 + 可信区间）。

## 六、实现方式：直接做基准（符号约束 + 弹性约束）

**直接实现 B3 基准识别 = 符号约束 + 供给弹性约束**（第三节），不绕递归。

- 结构脉冲响应的"公共机器"（用 Φ 迭代脉冲响应、投影到国别、算方差分解）是基准识别**本来就要建**的部分，会一起做好。
- 递归(Cholesky)**不是本文必需**：在我们的方案里它只是**稳健性的可选项之一**（与叙事约束、Känzig 工具、不同符号约束集并列，见 `论文指导/07`）。是否在稳健性一节加它，**等基准结果出来后再决定**（那时公共机器已在，几乎零成本）；也可以不加，用其它稳健性替代。
- 实现时我会在内部对"公共机器"做最简单的自测（写代码的常规手艺，不产出"递归识别结果"、不进论文）。

## 九、动手前需要你拍板的决策

| # | 决策 | 建议 |
|---|------|------|
| 1 | 弹性约束用不用、用什么上界？ | 用；上界 0.0258（Kilian-Murphy 2014） |
| 2 | 每份后验保留几个合格旋转？ | 先各留 1 个（标准做法），够用 |
| 3 | 旋转搜索上限多少次？ | 1000 次/份，报告接受率 |
| 4 | 要不要在 B3 里就做历史分解？ | 可先不做，出主结果后再加 |
| 5 | 稳健性识别用哪些？（**基准出来后再定**） | 递归/叙事/Känzig/不同符号集，任选，**非必需** |

> **这些设定的文献依据（速查，详版见 `论文指导/07` 第五节）：**
> - **弹性上界 0.0258**：Kilian-Murphy (2012《光有符号约束还不够》/2014)；需做敏感性（0.025 / 0.04 / 无边界）并主动回应 Baumeister-Hamilton (2019) 的"硬边界 vs 信息先验"之争。**技术红线**：数据标准化过，算弹性必须先还原原单位，且 `oil_prod`/`loil_price` 需为对数口径。
> - **每份留 1 个旋转**：RWZ (2010)、ARW (2018) 的标准算法（联合后验）；用"声明集识别 + 报告可信区间"回应 Fry-Pagan (2011) 的多模型批评。
> - **报告接受率**：Uhlig (2005)、ARW (2018) 惯例；1000 次只是计算上限，规范性来自透明披露。
> - **历史分解**：Kilian (2009)、Kilian-Lütkepohl (2017) 标准工具，先不做仅为排期。
> - **贯穿声明**：本文识别为**集识别**、推断依赖旋转先验（Baumeister-Hamilton 2015），故报告可信区间 + 敏感性。
> - **⚠️ 本文油市块无库存**：基准为 Kilian-Murphy (2012) 三变量式，②总需求/③油价特定需求的分离靠 **IGREA 的符号**（详见 `07` §5.5）；加库存重做为可选拓展。

## 十、复杂度与风险（诚实说明）

| 风险 | 说明 | 对策 |
|------|------|------|
| 符号约束接受率过低 | 约束太紧 → 很少有 Q 通过 | 放宽弹性上界、增旋转次数、先内部自测公共机器 |
| 弹性单位换算错 | 数据标准化过，弹性要还原原单位 | 用保存的均值/标准差还原，单独验证 |
| 三类冲击"贴标签"歧义 | 多个列可能都满足某模式 | 用符号+弹性联合约束缩小；必要时加叙事约束 |
| 部分识别的解读 | 只识别 3 个油价冲击、其余不解释 | 论文里说明"部分识别"，聚焦油价 |

> **总结**：B3 概念上是全项目最"绕"的一步，但它是**在 B2 后验上加一层后处理**，代码量有限、直接做基准（符号约束+弹性约束）即可。理解了第三节的"Ω = Cholesky×旋转，用油市块符号筛旋转"，就掌握了 B3 的灵魂。
