# load_data_oil.m 撰写说明（设计方案 + 与作者代码的区别）

> 目的：让你**完全看懂**这个文件在做什么、为什么这样写、和作者的 `load_data.m` 差在哪。

---

## 一、这个文件的职责（一句话）

**把 `all_data.xlsx` 里的两个表（EA 国别 / Global 全球）读进来，整理成主程序 `QFAVAR_GIRF.m` 需要的那一组变量，然后交回去。** 它是"数据"和"模型"之间的翻译官。

主程序只有一行调用它：
```matlab
[x,xlag,T,n,k,g,ng,dates,names,namesg,tcode] = load_data_oil(inflindx,outpindx,standar,r,p,interF,quant,dfm);
```

---

## 二、它返回的每样东西是什么（输出逐个说明）

| 返回值 | 含义 | 本文取值 |
|--------|------|----------|
| `x` | 国别数据矩阵（已标准化） | 248×64（月 × 国别序列） |
| `xlag` | `x` 的一阶滞后（仅 AR1x=1 时用） | 248×64（基准不用） |
| `T` | 时间长度 | 248 |
| `n` | 国别序列数 | 64（=4 变量×16 国） |
| `k` | VAR 每个方程的参数个数 | (4×3+5)×2 = 34 |
| `g` | 全球数据矩阵（已标准化） | 248×5 |
| `ng` | 全球变量数 | 5 |
| `dates` | 日期字符串 `'2005Jul'…` | 248×1 |
| `namesX` | 国别变量名 `'CPI.BR'…` | 64×1 |
| `namesYE` | 全球变量名 `'oil_prod'…` | 1×5 |
| `tcode` | 变换代码 | 全 1（无变换） |

## 三、撰写方案（每个关键决定，为什么这么做）

1. **用 `readtable` 读，而不是作者的 `xlsread`。**
   `xlsread` 是老函数，对"日期列 + 文本表头"混排容易出错；`readtable` 能自动把第一列当日期、把表头当列名，更稳、更好懂。

2. **列名用 `'VariableNamingRule','preserve'` 保留。**
   这样 `CPI_BR`、`NEER_US` 等原样保留，不被 MATLAB 改成 `Var1`。

3. **不做任何平稳化变换（这是最重要的一条）。**
   作者会按 `tcode` 调用 `transxFAVAR` 把原始指数做差分/取对数变平稳。**我们的数据在 Python 端已经是同比/增长率（平稳）**，再变换就错了。所以这里**跳过变换**，把 `tcode` 全设为 1（"无变换"）。

4. **日期转成 `'yyyyMMM'` 字符串（如 `'2005Jul'`）。**
   因为主程序里用 `datetime(dates,'InputFormat','yyyyMMM')` 解析画图横轴，我们喂它这个格式，主程序那行就**不用改**。

5. **变量名把下划线换成点：`'CPI_BR'→'CPI.BR'`。**
   主程序用 `extractBefore(names,'.')` 从名字里取"变量名"（CPI/IP/RATE/NEER），需要以 `.` 分隔。这样主程序取名逻辑**不用改**。

6. **`xlag` 不丢观测。**
   作者做滞后时会丢掉第一行（少一个月）。我们数据已平稳、不需要在这里差分，所以**保留全部 248 个月**，`xlag` 首行用自身占位（因为基准 AR1x=0，这块内容根本不参与计算）。

7. **标准化完全沿用作者逻辑。**
   `standar=1` 只标准化 `x`；`standar=2` 连 `g` 一起标准化（`zscore`）。主程序默认 `standar=2`，我们照用——数据我们给的是**未标准化**的，标准化交给这一步。

## 四、与作者 `load_data.m` 的逐点区别

| 方面 | 作者 `load_data.m` | 我们 `load_data_oil.m` | 为什么 |
|------|--------------------|------------------------|--------|
| 读取函数 | `xlsread`（旧） | `readtable`（新、稳） | 更好懂、少出错 |
| **平稳化** | 有 `tcode` 行 + `transxFAVAR` 变换 | **不变换**，`tcode=1` | 我们数据已平稳 |
| 通胀口径 | `inflindx` 选 总/去能源/核心 | 不用（只有一种） | 我们只做整体 CPI |
| 产出口径 | `outpindx` 切换 失业率↔工业生产 | 不用（直接工业生产） | 我们固定用工业生产 |
| 选列/删列 | 删 NPCR、`x(:,1:45)` 等 | 不用（64 列全要） | 我们变量已选定 |
| 丢观测对齐 | 因变换丢 1 行，`g` 也随之丢行 | **都不丢**，x/g 均 248 行 | 不在此处变换 |
| 去离群 | `adjout(g(:,1),...)` 处理第一个全球变量 | 默认不做（可选） | 我们油市数据已处理 |
| 维度 | n=45, ng=4 | **n=64, ng=5** | 16 国、油市块 |
| 硬编码国家数 | 代码里写死 9（欧元区 9 国） | **无**（直接读列名） | 更通用、不易错 |

> 一句话：**作者那套的一半代码是"选口径 + 平稳化 + 删列"的欧元区专用逻辑；我们数据已备好，这些统统不需要，所以我们的版本更短、更干净。**

## 六、怎么验证这个文件读得对（你在 MATLAB 里的自查）

运行主程序前，可以先单独测一下这个函数。在 MATLAB 命令窗口输入：
```matlab
[x,xlag,T,n,k,g,ng,dates,namesX,namesYE,tcode] = load_data_oil(1,2,2,4,2,0,[.1 .5 .9],0);
size(x)          % 期望: 248  64
size(g)          % 期望: 248  5
T, n, ng, k      % 期望: 248, 64, 5, 34
namesX(1:3)      % 期望: 'CPI.BR' 'CPI.CA' 'CPI.CN'
namesYE          % 期望: 'oil_prod' 'IGREA' 'loil_price' 'GSCPI' 'GEPU'
dates(1), dates(end)   % 期望: '2005Jul' , '2026Feb'
any(isnan(x(:))) , any(isnan(g(:)))   % 期望: 0  0 (无缺失)
```
> 说明：这里标准化后的 `x`、`g` 数值是"标准差单位"（均值 0、标准差 1），所以不要拿它和原始数值对比；对比"形状、名字、日期、无缺失"这几项即可。
